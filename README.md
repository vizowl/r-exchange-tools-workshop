# R Exchange tools workshop

You want to see if R and other open source tools can help solve some of the
challenges your analytics or data team are facing - but where and how do you get
started?

The list of barriers and decisions can seem overwhelming:

- What tools do I need?
- Where do I even __write__ code?
- What do I need to ask IT for permission to install?
- Is there a way to test out a minimal set of tools before talking to IT?
- What is a minimal set of tools?
- How do I work with the rest of my team?
- How can I integrate R into my wider organisations workflow?
- How do I share the results of my analysis securely?

## The minimal starting requirements

- A way to run R (or python)
- A way to edit your code
- A way to manage (share and store) your code

## Version control

Technically you don't need a way to manage your code to get started - 
so what is version control?

- Avoid code.v3-final.1.actually-final.signedoff.R
- Avoid "I'm about to jumping into the main file on the shared drive - just checking that no-one is currently editing it"
  - Followed by - "I missed that message and you just deleted 3 days work"
- Avoid "What version of the code did we run to provide the July report?"
